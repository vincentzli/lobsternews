# **Tokenmaxxing 危机：当 AI 烧掉的钱超过工程师薪资，Uber 与微软正仓促回归“人力优先”工程**

硅谷的“算力成本泡沫”终于在 2026 年初轰然破碎，其余震正剧烈冲击着科技巨头的决策层。我们正式进入了 **Tokenmaxxing（Token 极致扩张）** 时代——这是一种由工程师高强度使用 AI Agent（智能体）引发的畸形现象。它产生的 API 账单不仅超出了云端预算，甚至已经开始透支人类员工的薪资总额。

**Uber 的“价值鸿沟”内战**
根据 Uber 内部泄露的报告，首席技术官（CTO）Praveen Neppalli Naga 与首席运营官（COO）Andrew Macdonald 之间正爆发一场激烈的“代理人战争”。Naga 此前激进推行“Agent 优先”的工程文化，却意外陷入了内部人所称的“智能体疲劳”泥潭。Uber 工程师通过递归 Agent 循环来重构遗留代码，结果在短短不到五个月的时间里，就烧光了原本预定给 2026 全年的 AI 算力配额。

核心症结在于“价值鸿沟”。尽管这些 Agent 在进行 Tokenmaxxing——消耗数以亿计的 Token 来追求边际性的代码优化，但实际的产品价值并没有发生质变。正如红杉资本的 David Cahn 所言：“AI 基础设施建设所隐含的收入预期与实际收入增长之间存在巨大的断层。”在 Uber，这个断层已演变成一道深渊。据悉，Macdonald 正在强推一套“人类介入”配额制度以止住出血点，这实际上是给工程师能外包给 Agent 的“推理周期”设定了硬性上限。

**微软的转向：从“顶配智能”到“廉价智能”**
微软近期对 Claude Code 等第三方顶尖 Agent 的冷处理，标志着“不计代价追求前沿模型”时代的终结。现在的技术瓶颈不再是能力够不够，而是 **CpT（单项任务成本，Cost-per-Task）**。

Anthropic 首席执行官 Dario Amodei 曾发出警告：“如果我的收入达不到万亿美元规模，那么我购买海量算力的行为将不可避免地导致破产。”微软显然听进去了。通过退回到自主研发、性能“够用就好”的小模型，微软正试图解决“推理税”难题。行业已经清醒地认识到：如果一个 AI Agent 需要消耗 200 美元的 Token 才能修复一个价值 50 美元的 Bug，那么它就不是生产力工具，而是资产负债表上的毒药。

**算力泡沫的轮回**
我们正在目睹 2010 年代初云支出危机的 2026 年重演版。当年，企业意识到自己在为闲置服务器买单；今天，他们意识到自己在为“数字幻觉循环”付费。“AI 原生员工”模型只有在 Token 成本再降 90%，或者“Agentic ROI（智能体投资回报率）”真正拉动营收增长时才具备可持续性。在此之前，那些沉迷于 Tokenmaxxing 的信徒们，已经被列入了裁撤名单。
