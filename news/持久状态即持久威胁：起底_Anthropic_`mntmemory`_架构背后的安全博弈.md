# **持久状态即持久威胁：起底 Anthropic `/mnt/memory/` 架构背后的安全博弈**

AI 的“无状态”时代宣告终结。随着 Anthropic 发布托管 Agent（Managed Agents），整个行业正加速向基于文件系统挂载的持久化模型转型，其核心便是 `/mnt/memory/` 目录。对于开发者而言，这无异于 Agent 效用的“圣杯”：它让 Claude 能够跨越互不相干的会话，记住用户偏好、项目结构和复杂的中间状态，而无需承受外部数据库带来的巨大开销。

在实现细节上，Anthropic 选择使用标准文件系统挂载点而非不透明的 API 封装，堪称开发者体验（DX）层面的神来之笔。Agent 利用熟悉的 CLI 工具与之交互，这在内部形成了某种“草稿本”效应，使其能够自我纠错并梳理逻辑思路。然而，这种技术上的优雅掩盖了结构性的脆弱。

安全研究员 Elena Cross 指出：“我们正在构建一个可能被‘煤气灯操纵’进而导致自我破坏的系统。”其中的核心风险在于“记忆投毒（Memory Poisoning）”。与以往随窗口关闭即重置的提示词注入不同，存在于 `/mnt/memory/` 中的恶意文件更像是一种“驻留病毒”。只要攻击者能诱导系统完成一次恶意的写入，这种负面影响就可能贯穿该账户的整个生命周期。

这一趋势引发了科技界关于“LLM OS（大模型操作系统）”安全性的剧烈分歧。一方面，以 Andrej Karpathy 为代表的远见者将其视为必然进化，即模型扮演内核，而 `/mnt/memory/` 则是受其管理的磁盘；另一方面，Simon Willison 等批评者直言，这严重违背了安全系统设计的核心准则：指令与数据的分离。当 Agent 读取记忆时，它往往无法分辨哪些是参考数据，哪些是应当执行的潜在指令。

尽管 Anthropic 首席执行官 Dario Amodei 已经在近期研究中承认了这些“务实且具有毁灭性”的威胁，并紧急引入“不可变版本控制（Immutable Versioning）”作为临时补救措施。但随着 Agent 生态的扩张，维护“记忆卫生”的沉重负担最终将落在用户肩上。在硬核科技圈的审视下，结论已然明晰：我们正在用会话层面的安全性去换取 Agent 的生产力，而“遗忘”的代价从未像今天这样昂贵。
