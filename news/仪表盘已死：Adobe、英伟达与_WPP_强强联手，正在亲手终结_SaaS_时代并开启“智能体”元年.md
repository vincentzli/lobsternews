# **仪表盘已死：Adobe、英伟达与 WPP 强强联手，正在亲手终结 SaaS 时代并开启“智能体”元年**

拉斯维加斯见证了一个时代的落幕。过去十年，SaaS 的代名词是“仪表盘”，是人类点击按钮、处理数据的中转站。但在本周的 Adobe Summit 上，这个定义被彻底粉碎。Adobe、英伟达（NVIDIA）与 WPP 达成的深度联盟，标志着全球科技产业从“辅助型 AI”向“智能体营销”（Agentic Marketing）的决定性转向——软件不再仅仅等待指令，而是开始自主实现商业目标。

**编排者：Adobe CX Enterprise Coworker**
此次变革的核心是 Adobe CX Enterprise。这并非简单的产品更迭，而是将 Experience Cloud 彻底重构为一套 AI 原生操作系统。其最前端的交互层被称为“CX Enterprise Coworker”，一个构建在 Adobe 体验平台（AEP）之上的自主智能体层。

与传统的聊天机器人不同，Coworker 是一个具备“编排”能力的自主执行者。通过接入实时 CDP（客户数据平台）和客户旅程分析工具，它可以将“优化北美 Gen-Z 用户的忠诚度漏斗”这类抽象目标，拆解为一系列可执行任务：自动化受众分群、驱动 Firefly 生成视觉素材，并跨渠道部署营销方案。Adobe CEO Shantanu Narayen 指出：“我们正在重新架构 SaaS，使其参与到工作本身的重新定义中。”

**安全沙盒：NVIDIA OpenShell**
在企业环境中，自主行为的最大障碍始终是安全。如果无法为 LLM 访问客户数据和媒体预算提供“保险箱”，智能体就无法落地。

NVIDIA OpenShell 正是为此而生。这是一个安全、开源的运行时环境，充当了智能体的“沙盒”。它在网络和 API 层面强制执行基于策略的护城河，确保智能体仅在其定义的“爆炸半径”内行动。英伟达 CEO 黄仁勋表示：“通过 OpenShell，我们为智能体提供了一个既像银行金库一样安全，又像人类同事一样灵活的环境。”

**互操作协议：MCP 与 A2A**
为了避免出现“智能体孤岛”，该联盟正在力推两项关键技术标准：
1. **模型上下文协议 (MCP)**：由 Anthropic 开发并由 Adobe 大规模采用。MCP 允许 Adobe 营销智能体原生运行在任何企业级 LLM（如 ChatGPT、Gemini、Claude）中。这意味着 Adobe 的私有数据成为了所有智能体交互的“事实来源”。
2. **智能体对智能体 (A2A)**：该协议允许不同领域的智能体进行“谈判”。例如，WPP 的“媒介智能体”可以直接请求“创意智能体”为表现不佳的细分市场生成特定的广告变体，整个过程无需人类干预。

**品牌工程：从提示词到“深度微调”**
在技术底层，Adobe AI Foundry 与 NVIDIA NeMo 的结合解决了品牌一致性的难题。为了确保大规模自动化下的品牌调性，团队正在从简单的 Prompt 工程转向“深度微调”（Deep Tuning）。

通过利用 PEFT（参数高效微调）技术——特别是 LoRA 和 P-tuning，品牌可以将自身的 IP（Logo、配色方案、语调指南）直接植入模型权重。这里最具挑战性的是“IP 隔离”：确保品牌的私有数据不会泄露到基础模型中。英伟达通过 NIMs（推理微服务）将这些微调后的模型封装在安全、低延迟的容器中，实现了数据主权的绝对掌控。

**市场剧变：生产层的“蒸发”**
人类工作流的冲击波正由 WPP 引领。CEO Mark Read 已将这家全球最大的广告传播集团重新定义为一家“智能系统公司”，通过 WPP Open 平台部署全球化的专项智能体集群。

在 X.com 和 Reddit 的开发者社区中，共识已经形成：营销的“生产层”正在蒸发。Reddit 上一条高赞热帖写道：“体验云已死。Adobe 正在赌两年后我们不再需要 UI，我们只需要和 Coworker 说话。”

“智能体网络”（Agentic Web）不再是路线图上的虚线，而是在 MCP、OpenShell 和英伟达算力堆栈支持下已经落地的现实。
