# 解密 Jalapeño：OpenAI 与博通联手打造的 3nm ASIC，如何凿穿英伟达 Blackwell 的推理护城河？

2026年6月24日，定制芯片（Custom Silicon）领域跨越了一道分水岭。OpenAI 与博通（Broadcom）正式对外公布了“Jalapeño”——一款专为大语言模型（LLM）推理服务工作负载量身定制的专用集成电路（ASIC）。该芯片采用台积电最先进的 3nm 工艺制程，计划于 2026 年底前部署至超大规模数据中心。这也是 OpenAI 摆脱硬件依赖、走向硬件自主的“第一枪”。

这次发布不仅仅是多了一颗芯片，更是整个 AI 基础设施产业结构性转向的信号。随着 LLM 从早期的学术探索走向大规模的 AI Agent（智能体）商用落地，工程界的核心挑战已然从“模型训练”转移到了“模型推理”。而这一范式转移，彻底暴露了通用 GPU 架构与自回归 Token 生成算法之间的天然错配。

### 技术分水岭：通用 GPU 与原生推理 ASIC 的对决

要理解 OpenAI 为何要与博通联合开发定制化芯片，首先必须直面现代 GPU 在推理上面临的架构瓶颈。英伟达的 Blackwell（GB200）架构无疑是通用深度学习加速领域的巅峰之作——它拥有恐怖的单精度浮点算力，支持极低精度的 FP4/FP8 数据类型，并且通过高带宽的 NVLink 5 互联将数千个计算节点紧密相连，是训练工作负载的终极利器。

然而，大模型推理（LLM Serving）的运行逻辑与模型训练截然不同。自回归的 Decoder-only 模型需要逐个 Token 进行生成，这个过程被分为两个截然不同的执行阶段：Prefill（预填充/首字延迟）和 Decode（解码生成）。

* **Prefill 阶段**：模型并行处理输入的提示词（Prompt）。这一阶段是算力受限（Compute-bound）的，能非常高效地榨干通用 GPU 那庞大的 FLOPS（每秒浮点运算次数）。
* **Decode 阶段**：模型需要串行地生成后续的每一个 Token。每生成一个 Token，处理器都必须从内存中将整个模型的权重以及当前会话的 KV Cache（键值缓存）加载到寄存器中，执行计算，然后再将更新后的缓存写回内存。

在 Decode 阶段，算法的算术强度（Arithmetic Intensity，即计算操作与内存访问的比率）极低。这导致 Blackwell GPU 上庞大的 Tensor Core（张量核心）在绝大多数时间内都处于闲置状态，只能干等数据从 HBM（高带宽内存）中传输过来。

“英伟达的 Blackwell 是个怪兽，但它被设计得像一辆全能坦克——训练、推理、物理模拟、图形处理无所不能，”Tenstorrent 首席执行官 Jim Keller 对此直言不讳。“如果你只需要运行 Transformer 模型，你根本不需要一辆坦克。你需要的是一个快速、精简的数据流架构，能够把数据直接从内存推送到计算单元，而不是被复杂的调度和控制开销拖累。”

Jalapeño 正是通过放弃通用浮点运算性能，转而针对数据搬运与网络路由进行极限优化来解决这一痛点的。该芯片采用了一个接近光罩尺寸（Reticle-size）的巨型硅结构，集成了 6 个 HBM 模块。通过将宝贵的晶圆面积分配给高带宽内存接口和定制的路由通道，而非 GPU 中常见的通用执行单元和传统硬件调度器，Jalapeño 将“内存带宽-算力比”推向了极致。这确保了在逐个 Token 生成的 Decode 阶段，芯片的计算引擎能够始终处于饱和状态。

### 9 个月交付 Tape-Out：AI 辅助 EDA 引发的业内交锋

半导体工程界目前争议最大的焦点，在于 Jalapeño 的研发速度。在台积电先进的 3nm 工艺节点上，要交付一款接近光罩物理极限（Reticle-limited）的超大芯片，通常需要经历 18 到 24 个月极度严苛的物理设计、时序收敛与流片验证。然而，OpenAI 与博通却在短短 9 个月内创纪录地完成了流片（Tape-Out）。

硬件工程社区的怀疑论者对此深感忧虑。批评人士指出，跳过传统的、多阶段的物理验证步骤可能会严重危及台积电的晶圆良率，并为芯片在数据中心长期部署后的可靠性埋下隐患。历史经验表明，仓促进行物理版图（Layout）设计极易在长期运行中导致制造缺陷、局部热点（Hotspots）或电迁移（Electro-migration）失效。

对此，OpenAI 将开发周期大幅压缩的功劳归结于其自研的生成式大模型——即通过 AI 模型自动进行物理版图规划和路由设计，实现“用 AI 设计 AI 硬件”。

“硬件设计本质上是一个搜索空间优化问题，”行业分析机构 SemiAnalysis 的首席分析师 Dylan Patel 解释道。“通过引入强化学习（RL）模型来接管宏单元放置（Macro-placement）、电源网络规划以及时序收敛，OpenAI 与博通能够在几天内评估数百万种版图变体，而这在以前需要耗费数月。博通深厚且经过验证的 IP 库（包括其业界领先的 Tomahawk 交换芯片模块）提供了稳固的骨架，而 OpenAI 的模型则负责极限榨干版图空间。”

这种将 AI 驱动的电子设计自动化（EDA）与博通强大的工程执行力相结合的模式，虽然成功绕过了传统的版图设计瓶颈，但这些自动化规划的走线能否在数据中心高温、高电压波动的严酷环境中经受住时间的考验，仍需等待芯片实际部署后的反馈。

### 战略与市场变局：博通的崛起与英伟达的防线

Jalapeño 项目的战略暗流正在重塑整个半导体价值链。对 OpenAI 而言，其终极目标是通过垂直整合控制算力单位经济学。通过部署专为其自研推理内核（Inference Kernels）定制的 ASIC，OpenAI 宣称能将 Token 推理服务成本降低 50%。这将直接缓解运行 ChatGPT 及大规模智能体 API 服务所带来的高昂运营开销。

此外，此举也将弱化 OpenAI 对英伟达的资本开支依赖。Bernstein Research 资深分析师 Stacy Rasgon 指出这一动作的精妙平衡：

“开发定制芯片是超大规模云厂商试图从英伟达 70% 以上的高毛利中夺回利润的经典套路。但英伟达真正的护城河从来不是芯片本身，而是 CUDA 生态。对 OpenAI 来说，这一过渡相对平滑，因为他们早早布局了开源编程语言 Triton，它能直接编译并运行在定制硬件后端上。通过用 Triton 编写模型，OpenAI 得以绕过 CUDA 的掣肘，使 Jalapeño 这样的定制 ASIC 成为合格的‘即插即用’替代品。”

对于博通而言，Jalapeño 的成功流片进一步巩固了其作为科技巨头顶级定制芯片合作伙伴的地位。此前，博通已先后协助谷歌研发了多代 TPU，并与 Meta 联合开发了 MTIA 芯片。博通正凭借其商用 ASIC 业务筑起高墙。通过将博通标志性的 Tomahawk 交换芯片模块直接集成进 Jalapeño，系统级通信得以绕过英伟达私有的 NVLink 网络堆栈。这使得 OpenAI 可以直接利用标准且低成本的光纤以太网拓扑来构建超大规模集群。同时，系统集成商 Celestica 已接手单板与机柜级的设计，确保该芯片能无缝接入现有的标准化超算设施。

随着 Jalapeño 工程样片在实验室内开始跑起诸如“GPT-5.3-Codex-Spark”等测试工作负载，这场软硬件合流的战役已进入关键的验证阶段。如果 OpenAI 能成功在生产环境中将推理成本腰斩，这将向整个科技界证明：AI 算力的未来，终将属于特定领域定制的 ASIC，而非包揽一切的通用加速器。
