# 垂直整合还是慢性死亡？拆解 Figure AI 与 OpenAI “世纪离婚”背后的技术主权战

硅谷正在形成一个新的共识，而这个共识是对“AI 大于一切”信徒的一次公开叫板。Figure AI 本月宣布终止与 OpenAI 的深度合作伙伴关系，这绝非单纯的商业摩擦，而是一场具身智能领域的“独立宣言”。它标志着 Brett Adcock 推崇的“垂直整合（Vertical Integration）”哲学，正式向以 NVIDIA 和 OpenAI 为首的“通用大模型（Generalist Foundation Model）”阵营宣战。

这次决裂的导火索是冷酷的技术现实。当 Figure 尝试将 OpenAI 最新的多模态封装架构集成到 Humanoid 02 机器人时，他们撞上了一堵无形的墙：尽管高层逻辑推理近乎完美，但机器人的“反应神经”却极度迟钝。痛定思痛后，Figure 全面转向自研的“视觉-电机变换器（Visual-Motor Transformers, VMT）”。结果令人震惊：端到端延迟降低了 30%。在机器人领域，30% 的提升绝非边际改良，它直接决定了一个原型机是只能在实验室笨拙挪步，还是能真正进入工厂流水线。

Adcock 的逻辑简单而暴力：从“Token 到扭矩（Tokens-to-Torque）”的过程必须在设备端完成，必须基于硬件优化的架构。正如他最近所言：“如果把机器人的‘大脑’外包给通用 LLM 供应商，就像让一个视网膜延迟高达 200 毫秒的运动员去打职业网球。哪怕你智商两百，你也永远击不中球。”

这场分歧也在 X.com 上引发了顶级技术大腕的公开对垒。NVIDIA 的 Jim Fan 依然坚定为“物理 AI 基础模型”站台，押注通过 Project GR00T 进行的大规模仿真模拟终将让软件驾驭任何机械躯体。Fan 对 Figure 转型的评价言辞犀利：“垂直整合是一道沙筑的护城河。历史证明，这种专有的硬软件锁死方案，最终都会输给横向跨生态的规模化平台。”

然而 Figure 正赌上全部筹码，依靠其 5.8 万平方英尺的垂直整合工厂建立“数据壁垒”。通过掌控从定制执行器到第一视角视觉数据的完整闭环，Figure 正在生成模拟器根本无法复制的私有数据集。他们正在将具身智能从高层级的“推理”（系统 2）进化为超快速、高反应的“控制回路”（系统 1），将机器人视为一个单一、统一的神经网络。

市场的深层启示在于：如果 Figure 最终胜出，将证明纯 AI 软件公司在具身智能领域正面临被边缘化的风险。想要在物理世界取胜，你需要的不仅仅是权重和偏置，还需要一座工厂、一颗定制芯片，以及一个不需要等待 API 返回就能决定手指如何移动的 200Hz 高频控制环路。“壳公司（Wrapper）”的时代，在具身智能领域已经终结。
