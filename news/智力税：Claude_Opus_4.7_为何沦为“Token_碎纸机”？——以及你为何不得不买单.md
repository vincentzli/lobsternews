# 智力税：Claude Opus 4.7 为何沦为“Token 碎纸机”？——以及你为何不得不买单

4 月中旬，Anthropic 发布的 Claude Opus 4.7 在硅谷开发者圈层投下了一枚重磅炸弹。一边是令人窒息的统治力：它在 SWE-bench Pro 上跑出了破纪录的 64.3%，彻底夯实了 Anthropic 在智能体编码（Agentic Coding）领域的领头羊地位；而另一边，开发者们正集体声讨其隐藏在全新分词器（Tokenizer）架构下的“智力税”——API 成本飙升了 35%。

“AI 版缩量涨价”（AI Shrinkflation）正成为 X 和 Reddit 上的热议词。通过对 4.7 版本分词器的解构，我们发现 Anthropic 进行了一场激进的技术豪赌：牺牲 Token 密度来换取语义精度。与 4.6 版本相比，处理同样字符数的文本，新模型消耗的 Token 数量竟然增加了 1.35 倍。

从技术逻辑上看，这种更细粒度的子词切分（Subword Splitting）让模型能够以前所未有的深度“理解”复杂的代码库，但代价直接转嫁给了 API 用户。一家初创公司的创始人在 r/ChatGPTCoding 上直言不讳：“我们的自动调试业务毛利受到了毁灭性打击。我们原本是基于 4.6 版本的单位经济模型构建的，现在切换到 4.7，相当于在完全相同的产出下，被迫支付 35% 的附加费。”

Opus 4.7 的发布，也让那个神秘的内部代号“Mythos”（Capybara）再次浮出水面。据 3 月份泄露的信息，Mythos 在 SWE-bench Verified 上的得分高达惊人的 93.9%，但它目前仍被锁在针对顶级网络安全防御者的“Project Glasswing”计划中。

多位资深风投和行业观察者认为，Opus 4.7 实际上是 Mythos 的“安全蒸馏版”。Anthropic 强制模型输出推理链（Reasoning Chains），并将其透明化呈现。这确实显著提升了准确率，但也让模型变得极其臃肿且昂贵。

然而，“智力 vs 效率”的争论远不止于金钱。不少用户反馈，在超长上下文窗口中，模型出现了“煤气灯效应”（Gaslighting）。随着推理链的拉长，模型偶尔会在 200k 的上下文海洋中迷失，产生所谓的“上下文腐败”（Context Rot）。一位科技研究员指出：“它表现出一种高智商的自负——它能帮你修复一个零日漏洞，但转头就会信誓旦旦地跟你争辩，说你刚刚上传的那个文件根本不存在。”

不可否认，Opus 4.7 是目前星球上软件工程能力最强的模型，但天下没有免费的午餐。Anthropic 正在向市场抛出一个终极考题：为了那 10% 的逻辑推理增益，你愿意付出多大的成本代价？
