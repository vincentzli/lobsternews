# 跨越“内存墙”：拆解 Cerebras 晶圆级扩张与 Groq 200兆瓦豪赌，英伟达的推理垄断会被终结吗？

### 后训练时代的计算范式大转移

人工智能产业正在经历一场深刻的底层架构巨变。如果说 2020 到 2024 年的产业主轴是由大规模预训练（Pre-training）所定义——通过堆叠数以万计的通用 GPU 疯狂吞吐 Token 以炼制基座模型；那么站在 2026 年的前沿视域下，产业重心已不可逆转地转向了**测试时计算（Test-time Compute）**、实时推理闭环以及多智能体（Multi-agent）协作系统。

随着 OpenAI o 系列推理模型、Anthropic 的 Claude 智能体架构以及 Meta 开源推理集群的演进，前沿模型在给出最终回复前，往往需要在后台进行多达数百步的思维链（Chain-of-Thought）演算。这一转变直接导致 AI 数据中心的核心运行瓶颈，彻底从原始的 FP16/FP8 TFLOPS 算力峰值，转移到了**内存带宽（Memory Bandwidth）**与**单 Token 服务延迟经济学（Latency-per-token Serving Economics）**上。

在自回归大语言模型（LLM）的推理过程中，每生成一个 Token，都必须将多数十亿甚至数千亿参数模型的全部权重从内存中完整读取到计算逻辑单元一次。当面向单用户交互或紧密协作的多智能体闭环，且运行在单批次（$BS=1$）场景下时，传统通用 GPU 受制于外挂的高带宽内存（HBM）物理总线及非确定性的硬件动态调度器，会陷入极度的“内存受限”（Memory-bound）状态，导致大量计算算术逻辑单元（ALU）在等待数据传输时处于饥饿闲置状态。

```
+-----------------------------------------------------------------------------------+
|                                 AI 推理工作负载分类学                              |
+-----------------------------------------------------------------------------------+
|  高批次 / 吞吐量受限型（离线/批处理）        |  单批次 ($BS=1$) / 延迟受限型（实时交互）  |
|  - 海量文档索引与向量化处理                   - 语音交互 Agent 与实时对话 LLM      |
|  - 离线合成数据批量生成                      - 多智能体迭代推理与工具调用闭环     |
|  - 高并发量离线 Batch 批处理                 - 测试时计算（思维链 Chain-of-Thought） |
|  --> 最佳适配架构：HBM3e GPU（如 NVIDIA B200）--> 最佳适配架构：高密度 SRAM（Groq / CS-3）|
+-----------------------------------------------------------------------------------+
```

这一物理架构危机在半导体硬件领域引发了一场剧烈的分化。一边是以英伟达 Blackwell 架构为代表的传统 GPU 阵营，通过 Interposer 中介层在计算裸晶旁堆叠昂贵的 HBM3e；另一边，则是以 **Cerebras Systems** 和 **Groq** 为领军者、专为超低延迟推理而生的极客硬件架构。

随着 Cerebras 宣布在其位于加州米尔皮塔斯的 Flex（伟创力）超级工厂将 Wafer-Scale Engine 3（CS-3）的产能扩大 7 倍，以及 Groq 斩获 6.5 亿美元 E 轮融资、誓要在 2027 年底前建成 200 兆瓦（MW）算力规模的 AI 推理云，这场围绕万亿美元级 AI 推理市场的物理争夺战已正式演化为白热化的公开对抗。

### Cerebras CS-3：晶圆级封装、44GB 片上 SRAM 与 7 倍产能狂飙

在片上集成度的技术极端，是 Cerebras Systems 部署于 CS-3 系统核心的第三代晶圆级引擎（WSE-3）。传统的半导体制造工艺是将一块 300 毫米（12 英寸）的硅晶圆切割成数百个独立的芯片裸晶，而 Cerebras 则颠覆性地直接使用整块台积电 5nm 工艺打造的 300 毫米晶圆，来构建单一颗庞大的超巨型处理器。

#### CS-3 核心硬件规格拆解
* **晶体管总量：** 单颗晶圆集成高达 **4 万亿** 个晶体管。
* **计算核心：** 90 万个专为 AI 矩阵运算优化的可编程稀疏线性代数核心（SLAC）。
* **片上内存：** **44 Gigabytes** 分布式静态随机存取存储器（SRAM），与计算逻辑单元同存互连。
* **晶圆级内存带宽：** 达到震撼的 **21 Petabytes/s**（$21 \times 10^{15}\text{ bytes/sec}$）。
* **峰值计算性能：** 125 FP16 Petaflops。

通过将整个模型权重（或相当规模的流水线并行阶段）完整保留在片上 SRAM 中，CS-3 彻底抹平了传统处理器与存储器之间的物理鸿沟。其高达 21 PB/s 的内存带宽，大约是英伟达 H100 GPU（$3.35\text{ TB/s}$）的 2500 倍，更是英伟达 Blackwell B200（$8.0\text{ TB/s}$）的 2625 倍。

```

|                        内存带宽物理对比（对数刻度 Log Scale）                        |
+------------------------------------------------------------------------------------+
| 英伟达 H100 (HBM3):    |||| (3.35 TB/s)                                             |
| 英伟达 B200 (HBM3e):   |||||||| (8.0 TB/s)                                          |
| Groq LPU 机柜 (SRAM):  |||||||||||||||||||||||||| (聚合达 80 TB/s)                 |
| Cerebras WSE-3 (SRAM): |||||||||||||||||||||||||||||||||||||||||||||||||| (21,000 TB/s)|
+------------------------------------------------------------------------------------+
```

#### 晶圆级封装与 23kW 液冷工程挑战
实现晶圆级集成必须跨越半导体工程的两座大山：**晶圆良率缺陷** 与 **极致散热**。

1. **缺陷容忍与硬件冗余：** 在标准晶圆上，任何微小的晶格缺陷都会导致整块切割芯片报废。Cerebras 的解法是在晶圆设计中额外制造约 1.5% 的冗余核心，并引入硬件级缺陷收割机制。如果在晶圆测试中发现某个核心损坏，晶圆内部的可编程路由逻辑会自动绕过该故障点，将 2D 网格网络无缝重定向至相邻的完好核心。
2. **23 千瓦单机液冷系统：** CS-3 单台 15U 机架式服务器的功耗超过 23 千瓦，呈现出极高的热密度。传统风冷散热器在此毫无用武之地。Cerebras 联合设计了定制的液冷分流歧管，使冷冻水能够直接与紧贴整块硅晶圆表面的铜质冷板接触，进行高效热交换。

为了满足企业端对 CS-3 算力基础设施爆发式的需求，Cerebras 于 2026 年 7 月与代工巨头 Flex（伟创力）达成深度合作，在加州米尔皮塔斯的先进制造基地大幅扩展产能。这一合作将 CS-3 的生产能力直接**提升 7 倍**，并在硅谷本地建立了专用的自动化高密度供电测试站，以及特殊的液冷真空注流验证车间。

> *"现代计算最根本的瓶颈从来不是算力（FLOPs），而是在不同芯片封装之间通过铜线或光纤搬运数据的能量与延迟损耗，"* **Andrew Feldman**，Cerebras Systems 首席执行官兼联合创始人在工程主题演讲中谈到。*"当你把 44GB 的内存和 90 万个核心直接集成在同一块晶圆上时，你就不需要浪费电能把 Bit 数据通过 PCIe 或 InfiniBand 搬来搬去。一切都在以光速运转。晶圆级架构对于实时推理而言不是备选选项，而是物理上的刚性要求。"*

针对超过 44GB SRAM 容纳极限的超大模型（如 Llama 3 70B 或 405B），Cerebras 引入了 **MemoryX** 子系统。这一外置存储架构实现了计算与存储的解耦流式传输，使得单台 CS-3 或 CS-3 集群无需重写复杂的模型并行代码，即可支持高达 24 万亿参数的超大规模模型推理。

### Groq LPU 架构：确定性硅晶、静态调度与 200MW 算力新云

与 Cerebras 沿垂直方向探顶晶圆级极限不同，Groq 选择了沿着水平横向扩展的路径，通过基于 **LPU（Language Processing Unit，语言处理单元）** 的“编译器优先”架构来破局推理瓶颈。

Groq 由谷歌 TPU 初始团队的核心首席架构师 **Jonathan Ross** 创立。Groq 架构完全摒弃了传统 GPU 中复杂的硬件乱序执行、硬件分支预测器以及动态内存缓存等机制。

```
传统 GPU 执行模型（以英伟达 CUDA 为例）：
+--------------------+      +--------------------+      +--------------------+
| 硬件动态调度器      | ---> | 动态 Warp 线程执行  | ---> | Cache Miss / 延迟抖动|
+--------------------+      +--------------------+      +--------------------+
(内存延迟不可预测、硬件仲裁开销高、执行过程非确定性)

GROQ LPU 执行模型：
+--------------------+      +--------------------+      +--------------------+
| 静态 Groq 编译器   | ---> | 确定性时钟周期执行   | ---> | 零 Cache 抖动      |
|                    |      |                    |      | 精确延迟下限保障   |
+--------------------+      +--------------------+      +--------------------+
(编译期预先调度所有数据流动、零硬件仲裁开销、100% 确定性)
```

#### LPU 的核心技术特征
1. **确定性硬件执行（Deterministic Execution）：** LPU 芯片内部没有硬件调度器，也没有缓存控制器。每一个寄存器传输、矩阵乘法乃至芯片间的互连数据交换，全都在编译阶段由 Groq Compiler 做出精确到纳秒级的静态时间表规划。
2. **SRAM 优先的存储拓扑：** 每颗 LPU 芯片内置 230 MB 的片上 SRAM，配合内部 Crossbar 交叉开关，单节点可提供超过 80 TB/s 的内存带宽。
3. **线性互连扩展：** 由于硬件运行精确到纳秒级，Groq 芯片之间可以通过直连铜线直接通信，完全摆脱了高延迟的以太网/InfiniBand 交换机以及复杂的网络拥塞控制协议。

2026 年 6 月 22 日，Groq 宣布完成由 Disruptive 和 Infinitum 领投的 **6.5 亿美元 E 轮融资**。Groq 正将这笔资金全力投向其自营“Neocloud（新型算力云）”基础设施的建设中，目标是在 2027 年底前，将其全球 13 个数据中心的总供电规模扩展至 **200 兆瓦（MW）**。

此外，在 2025 年底与英伟达完成一项技术授权交易后，Groq 的推理软件栈正与其专属硬件平台（包含 LPX 模块化节点）深度融合，支持开发者在规模化数据中心内部署超低延迟推理流水线。

在运行 Llama-3-70B（$BS=1$）的实际基准测试中，Groq LPU 的生成速度达到了惊人的 **300 至 500 Tokens/秒/用户**，首字延迟（TTFT）控制在 15 毫秒以内，相较于标准 H100 集群呈现出数量级的领先优势。

> *"GPU 本质上是为并行矩阵数学改装的通用渲染引擎，"* Groq 首席执行官 **Jonathan Ross** 在 X.com 上点评道。*"当你在运行交互式 LLM 或每秒需要进行 50 次子调用的 Agent 软件时，每次响应让你等 200 毫秒就会直接破坏产品体验。Groq 打造的是一条确定性的流水线。编译器知道每一个 Byte 在精确到纳秒的时刻位于何处。这就是为什么别人在等 HBM Page Hit 的时候，我们可以把延迟做到 10 毫秒以下。"*

### 基础设施与经济学博弈：TCO、功耗墙与 CUDA 高墙

#### 1. SRAM 密度困境 vs. HBM 经济学
无论 Cerebras 还是 Groq，其面临的最核心技术与商业瓶颈，都是**静态随机存取存储器（SRAM）的物理面积扩展极限**。

SRAM 存储单元每 Bit 需要 6 个晶体管（6T SRAM），极其消耗硅片面积（在 5nm 工艺下，1 MB 的 SRAM 需占用约 1 $mm^2$ 面积）。相比之下，HBM3e 通过硅通孔（TSV）技术将动态存储裸晶（DRAM）在垂直方向 3D 堆叠，可以用极小的晶圆面积成本提供高达 192GB 的内存容量。

```
SRAM 存储单元（每 Bit 需 6 个晶体管 6T）：
  +---------------------------------------------------+
  | M1   M2   M3   M4   M5   M6  (极高晶圆面积成本)  |  --> 5nm 下约 1 MB / mm^2
  +---------------------------------------------------+

HBM3e 3D 堆叠（TSV 垂直贯穿）：
  +---------------------------------------------------+
  | DRAM 裸晶 8 (TSV)                                 |
  | DRAM 裸晶 4 (TSV)                                 |  --> 每颗 GPU 可挂载高达 192 GB
  | 底层 Logic 逻辑裸晶                                |
  +---------------------------------------------------+
```

知名半导体分析机构 *SemiAnalysis* 创始人 **Dylan Patel** 对此做出了中肯的商业分析：

> *"在内存带宽和延迟表现上，SRAM 是无可争议的王者，但它承受着极度昂贵的密度惩罚，"* **Dylan Patel** 指出。*"要纯靠 SRAM 容纳一个 4000 亿参数的 FP8 模型，Groq 需要把数千颗 LPU 芯片通过机柜互连，这极大拉高了硬件的资本开支（CapEx）。英伟达的 HBM 方案虽然在单批次 $BS=1$ 时单 Token 延迟较高，但其巨大的存储密度允许云厂商在单台机柜上并发服务数万个用户流。这本质上是‘单用户极致低延迟’与‘单美元总 Token 吞吐量’之间的商业博弈。"*

#### 2. 数据中心功耗墙（Power Wall）
全球数据中心正面临严峻的电力配额限制。传统高密度数据中心机架通常按 10 kW 到 15 kW 进行电力设计。

部署 Cerebras CS-3（单台 23 kW）或需数百张板卡构成的 Groq 机组，必须对数据中心基础设施进行彻底改建，引入片上水冷歧管、高压 CDU（冷冻液分配单元）以及 415V 交流配电系统。

#### 3. 软件生态护城河：CUDA 与 图编译器之困
英伟达真正的护城河从来不只是硅片本身，更是 **CUDA** 及其上层生态系统（**vLLM**、**TensorRT-LLM**）。

专用推理架构必须依赖专用的编译器：
* Cerebras 需要通过其 CSL（Cerebras System Language）与 SDK 重新编译编译流；
* Groq 则高度依赖 Groq Compiler 对模型的计算图进行全静态调度规划。

当开源 AI 模型架构快速迭代——例如引入动态路由的混合专家模型（MoE）、线性注意力机制（Mamba / RWKV）或 Diffusion-Transformer 混合架构时，定制图编译器往往难以像 PyTorch/CUDA 原生算子那样快速适应并实现高效的动态 Kernel 分发。

芯片设计传奇人物、Tenstorrent 首席执行官 **Jim Keller** 在社交平台上直言编译器的巨大挑战：

> *"造一块定制芯片固然很难，但写出一个能够对任意神经网络计算图进行静态调度、且完全不回退到慢速 CPU 降级执行的编译器，难度要高出十倍，"* **Jim Keller** 强调。*"如果你的芯片物理上依赖静态调度，那么每当 AI 研究员发明一种全新的注意力机制或动态路由算法时，你的编译器就会崩溃。2027 年能赢的硬件，必须在具备动态执行能力的同时，绝不丢掉底层的计算效率。"*

### 总结：AI 计算的双极化未来

半导体产业并不会演变为“赢家通吃”的单一硬件格局，而是加速分化为**双核驱动的二元对立体系**：

1. **预训练与高并发离线吞吐（Pre-Training & Batch Processing）：** 继续由搭载庞大 HBM3e/HBM4 内存池的通用 GPU（如英伟达 Blackwell/Rubin、AMD Instinct MI350）所主导，专为万亿参数基座模型的训练以及数百万级的离线合成数据批处理而优化。
2. **实时 Agent 推理与极速响应（Real-Time Agentic Serving）：** 由 Cerebras CS-3 和 Groq LPU 这类 SRAM 密集型、确定性执行的专用架构所驱动。随着企业级软件从被动的文本 Chatbot 演变为亚秒级响应的自主 Agent 系统——一个 Agent 必须在几秒钟内完成数十次工具调用——晶圆级与 SRAM 推理平台毫秒级的响应能力正从“奢侈品”变成“刚需”。

Cerebras 在米尔皮塔斯工厂的 7 倍扩产，以及 Groq 掷下 6.5 亿美元构建 200 兆瓦推理云，标志着专用推理硬件正式走出实验室试炼期。

硅晶战线已然划定：这是外挂 HBM 的海量容量，与片上 SRAM 的光速带宽之间的一场终极决战。
