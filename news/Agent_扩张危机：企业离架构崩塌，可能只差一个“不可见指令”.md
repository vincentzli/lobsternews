# **Agent 扩张危机：企业离架构崩塌，可能只差一个“不可见指令”**

在沙山路（Sand Hill Road）的投行办公室与帕洛阿尔托的工程中心，关于大模型的叙事已经发生了剧变。我们已经跨越了单纯的“模型军备竞赛”，直接进入了安德烈·卡帕斯（Andrej Karpathy）所预警的“烂泥末日”（Slopacolypse）。截至 2026 年 4 月，自主 AI 的蜜月期宣告终结。高德纳（Gartner）最新发布的 2026 年度报告证实了业界最深重的恐惧：随着自主 Agent 从实验性 Demo 涌入毫无防备的生产环境，高达 94% 的企业正处于“治理真空”之中。

这种现象被称为“Agent 扩张”（Agent Sprawl）。它是传统“影子 IT”的进化版，但危险程度呈几何倍数增长。与一个简单的影子 SaaS 应用不同，AI Agent 是具备主动性的实体：它能推理、能执行代码，更致命的是，它能自主衍生出子 Agent。

**身份认知的断层：传统 IAM 的终结**
传统的身份与访问管理（IAM）体系是为“人”设计的，依赖于静态会话和单一身份。然而，AI Agent 运行在“嵌套身份”之上。当一个主 Agent 将任务下放给三个子 Agent 时，现有的安全栈（如 OAuth 2.0 或 SAML）会瞬间丢失追踪链路。我们正目睹传统系统在验证“非人类实体”（NHIs）时的全面溃败。

里德·霍夫曼（Reid Hoffman）近期将此定性为“协议困境”。在数十亿 Agent 即将开启交互的时代，由于缺乏统一的治理框架，这场“认知工业革命”正处于无图航行状态。霍夫曼在简报中指出：“这触及了关于协议、执行力以及 Agent 权限边界的底层问询。”

**影子 Agent 与“不可见指令”矢量**
企业稳定性的最大威胁源自“影子 Agent”——那些由业务部门私自部署、绕过 IT 审批流程的 Model Context Protocol（MCP）服务器和 LLM 驱动工具。这些 Agent 往往拥有“上帝模式”权限，使其成为“间接提示词注入”（IPI）的完美猎场。

黑客的攻击手段已不再局限于简单的提示词工程，而是进化到了“ASCII 走私”（ASCII Smuggling）。利用 Unicode 标签区块（U+E0000–U+E007F），攻击者可以嵌入人类肉眼不可见、传统 DLP 扫描器无法识别，但 Agent 分词器却能解读为高优先级指令的代码。一段看似平常的网页摘要，就可能触发影子 Agent 窃取会话 Cookie 甚至清空云端环境。

**自主化的技术债**
尽管 Agent 工作流带来的生产力飞跃不容置疑，但其维护债正以机器速度堆积。缺乏监管的系统正在企业基础设施内部制造“暗逻辑”——那些没有任何人类能完全理解、更无法实时审计的黑盒工作流。

**转向“治理即服务”（GaaS）**
为了填补真空，行业的前沿阵地正向“治理即服务”（GaaS）转型。这要求架构进行根本性重组：
- 意图感知鉴权：从验证“密钥是否有效”转向验证“操作是否符合 Agent 的业务语境”。
- 实时熔断机制：部署自主断路器，一旦检测到行为异常，毫秒级吊销 Agent 身份。
- 集中式 GaaS 平台：为所有非人类实体提供统一控制平面，引入高德纳警告称目前 88% 的部署中都缺失的“人在回路”（HITL）监管。

正如萨姆·奥特曼（Sam Altman）所警告的那样：“AI Agent 正在成为一个麻烦。”他特别提到了 Agent 在识别零日漏洞方面恐怖的效率。如果你管理 Agent 的速度跟不上它们思考的速度，那么你管理的就不是一支劳动力，而是一个正在倒计时的定时炸弹。
