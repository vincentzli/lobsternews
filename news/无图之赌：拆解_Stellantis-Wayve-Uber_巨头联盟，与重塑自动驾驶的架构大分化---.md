# **无图之赌：拆解 Stellantis-Wayve-Uber 巨头联盟，与重塑自动驾驶的架构大分化**

---

### 三方联盟：战略范式的剧变

L4 级自动驾驶的商业化下半场已经轰然拉开序幕。2026年6月，全球汽车巨头 Stellantis、英国自动驾驶 AI 独角兽 Wayve，以及全球出行网络巨头 Uber 联合签署了一份里程碑式的战略谅解备忘录（MoU），宣布将在全球范围内共同打造、部署并规模化运营无驾驶员 Robotaxi 舰队。

与第一代自动驾驶（AV 1.0）——那种依赖百万美元级改装硬件、受限于极度僵硬且局部化的运行设计域（ODD）——截然不同，这个“三国联盟”将自动驾驶的价值链彻底拆解为三个专业化分工的高效层级：

```
+-----------------------------------------------------------------------+
|                             需求与车队层                               |
|                  Uber 全球需求聚合网络与智能调度系统                    |
+-----------------------------------------------------------------------+
                                   |
                                   v
+-----------------------------------------------------------------------+
|                            自动驾驶大脑层                             |
|          Wayve 无图端到端具身智能 Stack (VLA / GAIA 世界模型)            |
+-----------------------------------------------------------------------+
                                   |
                                   v
+-----------------------------------------------------------------------+
|                            硬件与整车底盘层                           |
|       Stellantis L4-Ready Platform™ (双冗余线控转向/制动架构)          |
+-----------------------------------------------------------------------+
```

1. **硬件与制造层：** Stellantis 提供原生量产的 **L4-Ready Platforms™**，在出厂前即装配好传感器阵列、容错供电架构以及双冗余线控转向（Steer-by-Wire）与线控制动（Brake-by-Wire）控制回路。
2. **软件与算法层：** Wayve 交付其无图 **具身智能（Embodied AI）** 软件栈——基于多模态视觉-语言-动作（VLA）大模型与生成式世界模型构建的端到端（E2E）深度学习架构。
3. **需求与商业化层：** Uber 将无人驾驶车队无缝接入其全球出行需求网络，彻底消除获客成本，并极大化提高 Robotaxi 在各大都市圈的资产利用率。

这一横向解耦联盟，直接向 Alphabet 旗下的 Waymo（自行研发软件、采购并改装捷豹 I-PACE/极氪硬件、自营 App）以及特斯拉（掌控制造、全栈 FSD 软硬件及 Cybercab 自营生态）所主导的全栈垂直整合范式发起了正面挑战。

### 架构大分化：AV 1.0 vs. AV 2.0

在这场联盟背后的核心，是一场自动驾驶工程哲学上的根本分歧：**依赖高精地图的模块化软件架构（AV 1.0）**，与 **无图端到端大模型架构（AV 2.0）** 的路线决战。

```
AV 1.0 管道（传统地理围栏模块化架构）：
[传感器] -> [感知/目标检测框] -> [高精地图匹配与定位] -> [行为状态机] -> [MPC轨迹规划] -> [执行器控制]
*(脆弱、高精地图维护成本高昂、跨人工规则模块边界产生累积误差)*

AV 2.0 管道（Wayve 具身智能架构）：
[多摄像头/LiDAR 视频流] ---> [深度神经网络 (VLA / GAIA 世界模型)] ---> [直接输出轨迹/动作 Token]
*(无图化、连续向量空间表征、在未映射过的复杂城市道路中具备零样本泛化能力)*
```

#### AV 1.0 的死局
传统自动驾驶架构将驾驶解耦为离散的、基于人工工程规则的子模块：感知（Perception）、定位（Localization）、预测（Prediction）、规划（Planning）与控制（Control）。这一管道极度依赖厘米级高精地图（HD Maps），图中精准标注了车道线、红绿灯映射关系、停止线、路沿高度等几何信息。

尽管 AV 1.0 在凤凰城、旧金山等受限地理围栏城市取得了令人瞩目的安全记录，但它面临着难以克服的结构性瓶颈：
* **维护成本高昂：** 高精地图需要高频更新。一次未预警的道路施工、临时改道或车道线磨损，都可能导致定位失效或危险的急刹行为。
* **规模化扩张受阻：** 将 AV 1.0 系统推向一座新城市，需要数月的测绘、手动标注与局部规则调校。
* **误差级联传递：** 跨模块边界的误差累积（例如感知模块中的误检框会直接污染下游的运动预测），迫使工程师必须编写数以万计的逻辑补丁（Edge-Case Rules）。

#### AV 2.0 的破局
Wayve 的 AV 2.0 范式废弃了人工撰写的流水线，改用在海量人类专家驾驶视频上进行端到端训练的深度神经网络。传感器数据（摄像头、毫米波雷达及可选的稀疏 LiDAR）被直接映射为轨迹向量、转向角与加减速指令。

Wayve 的具身基座模型不再将传感器输入与静态高精地图数据库进行比对，而是在潜空间（Latent Space）中直接学习动态空间表征、视觉几何关系与意图预测。

### 技术深潜：Wayve 的 AI 引擎 —— GAIA 与 LINGO

Wayve 的技术护城河建立在两大突破性架构之上：用于自动驾驶的生成式 AI（**GAIA**）与视觉-语言-动作模型（**LINGO**）。

```
Wayve 神经网络架构全景：
                   +----------------------------+
                   |    多传感器视觉输入数据流   |
                   +----------------------------+
                                  |
                                  v
+-----------------------------------------------------------------------+
|                       LINGO-2 VLA 模型主干                           |
|  - 视觉编码器：时空特征抽取                                            |
|  - 语言解码器：思维链 (CoT) 推理与可解释性 Token                       |
|  - 动作输出头：连续控制轨迹 Token (x, y, theta, v)                      |
+-----------------------------------------------------------------------+
                                  |
            +---------------------+---------------------+
            |                                           |
            v                                           v
+-----------------------+                   +-----------------------+
|  GAIA 世界模型        |                   |  STLA Brain 底盘执行  |
| 闭环仿真生成器        |                   |  转向与制动线控执行器  |
+-----------------------+                   +-----------------------+
```

#### 1. GAIA（Generative AI for Autonomy）：潜空间世界建模
为了在无需将车辆置于现实危险环境的前提下训练和验证驾驶策略，Wayve 开发了 **GAIA-1**（90亿参数）及其升级版 **GAIA-2**。

GAIA 充当了“物理世界仿真器”的角色，通过视觉提示、文本条件与主车动作 Token，预测未来的驾驶场景：
$$\mathcal{P}(S_{t+1:t+k} \mid S_{1:t}, a_{t:t+k}, T)$$
其中 $S$ 代表视频帧序列，$a$ 代表动作 Token（转向、制动），$T$ 代表自然语言提示词（例如：*“黄昏时分的强降雨天气，路边突然有行人横穿马路”*）。

通过自回归生成逼真的多视角驾驶视频，GAIA 实现了真正的闭环合成仿真。驾驶模型可以在生成的虚拟世界中自主导航，使工程师能够在无需实车场地测试的情况下，对极端长尾失效模式（如特种车辆强行加塞、恶劣气候）进行大规模验证。

#### 2. LINGO：多模态视觉-语言-动作（VLA）模型
端到端神经网络长久以来被诟病为无法解释的“黑盒”。为了破局，Wayve 推出了 **LINGO-1** 与 **LINGO-2**。其中 LINGO-2 是一个视觉-语言-动作模型（VLAM），将视觉理解、文本推理与驾驶动作输出统一在同一个自回归 Transformer 架构之中。

在推理阶段，LINGO-2 实时处理多路视频流，并同步输出两大内容：
1. **动作输出 Token（Action Tokens）：** 连续的数值轨迹向量 $(x_t, y_t, \theta_t, v_t)$，直接下发至车辆运动控制器。
2. **解释性语言 Token（Language Tokens）：** 输出自然语言实时解说，阐明模型的推理过程（例如：*“正在避让迎面驶来的自行车，随后在未标注路口执行右转”*）。

这种自然语言接口为安全工程师提供了前所未有的可解释性，允许安全审计人员在长尾极端事件发生时，直观审查神经网络内部的注意力机制与决策逻辑。

#### 零样本城市泛化能力
由于 Wayve 的模型学习的是通用的视觉先验，而非对特定地理几何进行“死记硬背”，因此展示出了极其出色的零样本（Zero-Shot）与少样本迁移能力。在伦敦、东京及多个美国大都市区的测试中，Wayve 的基座模型在极少微调甚至无需微调的情况下，即可轻松应对狭窄巷道、通行规则切换（英国左行 vs. 美国右行）以及未见过的交通信号灯设计。

### Stellantis 的硬件底座：L4-Ready Platforms™

再顶尖的 AI 软件，若没有能够实现“零人类干预”容错的整车硬件也无从谈起。Stellantis 的 **L4-Ready Platform™** 正是为此而生，它在其 STLA 汽车架构中硬核集成了整车级硬件冗余。

                |                                      |

|    主线控转向系统 (SbW)        |      |    后备线控转向系统 (SbW)      |

+-------------------------------+      +-------------------------------+
                |                                      |

#### 关键硬件规范：
1. **线控转向（SbW）与线控制动（BbW）冗余设计：**
   * 取消机械转向柱，全面转向双冗余数字执行器。
   * 采用独立的双电源供电（12V/48V 分立架构）与双绕组电机设计。若主转向电机绕组发生故障或过热关断，后备绕组将在 **<10毫秒** 内无缝接管，确保控制扭矩零丢失。
   * 采用电液主制动系统，并搭配独立的电子驻车（EPB）执行器，确保在硬件出现严重故障时依然能够执行安全制动轨迹。

2. **STLA Brain 集中式计算架构：**
   * 采用集中式面向服务架构（SOA），实现底盘硬件执行器与上层 AI 应用软件的解耦。
   * 搭载 10 Gbps 高吞吐量车载以太网主干网，将未经压缩、超低延迟的摄像头与传感器数据流直接传输至 Wayve 的推理算力单元。

3. **车规级原生工厂集成：**
   * 放弃了后装时代的“车顶大包”改装方案，Stellantis 在工厂流水线组装阶段，就将摄像头、短距毫米波雷达及近场传感器完美嵌入车身。
   * 自带传感器清洗系统（加热喷水嘴与声波滤镜自清洁），这是全天候商业化运营的硬性刚需。

### 安全与监管验证：黑盒 AI 的终极难题

将端到端神经网络推向无驾驶员的商业化运营，在监管与功能安全层面构成了前所未有的挑战。

```
安全与监管验证对比矩阵：
+------------------------+------------------------------------+-------------------------------------+
| 标准规范               | 传统基于规则模式 (AV 1.0)           | 端到端深度神经网络模式 (AV 2.0)     |
+------------------------+------------------------------------+-------------------------------------+
| ISO 26262              | 形式化状态机验证；                 | 概率型失效模式；                    |
| (功能安全)             | 确定性代码路径隔离。               | 必须搭载硬件级确定性安全护栏。       |
+------------------------+------------------------------------+-------------------------------------+
| ISO/PAS 21448          | 硬编码的长尾规则覆盖               | 依靠 GAIA 世界模型                  |
| (预期功能安全 SOTIF)   | 与物理封闭场地测试。               | 进行合成分布覆盖测试。              |
+------------------------+------------------------------------+-------------------------------------+
| 安全案例审计           | 代码审查与                         | LINGO 自然语言推理解说              |
| 与可解释性             | 离散决策模块的单元测试。           | 与潜空间注意力可视化。              |
+------------------------+------------------------------------+-------------------------------------+
```

#### 功能安全（ISO 26262）与 SOTIF（ISO/PAS 21448）
传统汽车安全标准如 **ISO 26262** 均建立在“确定性软件执行路径”的假设之上，即每个子模块都可以对照明确的需求规范进行单元测试。然而，深度神经网络本质上是概率模型。

为了满足严格的功能安全合规要求，联盟采用了混合架构：
* **DNN 规划器（Planner）：** 利用端到端权重，生成拟人化、高舒适度的最优驾驶路径。
* **确定性安全护栏（Deterministic Safety Shield）：** 一个运行在独立计算核上的轻量级、基于物理规则的安全校验层。该安全护栏硬性注入了物理约束（如最大制动减速度、碰撞时间 TTC 阈值、道路边界约束）。一旦神经网络输出的轨迹违背了这些物理安全底线，确定性安全护栏将瞬间剥夺控制权并执行安全停车。

#### 如何向监管证明“黑盒”是安全的？
向监管机构（如美国 NHTSA、英国交通部或欧盟监管方）证明没有硬编码规则的神经网络同样安全，主要依赖两条路径：
1. **基于合成覆盖率的统计学验证：** 利用 GAIA 等世界模型生成数十亿公里的虚拟仿真行驶里程，在长尾天气与复杂交通流分布中进行概率化的安全收敛验证。
2. **自然语言审计追踪：** 在车队测试中利用 LINGO-2 的实时自然语言解说流，验证模型的视觉注意力机制是否精准识别到了潜在危险要素。

### 商业模式与行业经济学：横向平台 vs. 垂直整合

L4 级自动驾驶的商业化落地，演化出了两条针锋相对的商业路径：

```
商业模式对比矩阵：

        横向平台模式 (三方联合)                              垂直整合模式
  (Stellantis + Wayve + Uber 架构)                     (Waymo / Tesla Cybercab 架构)

+----------------------------------+               +----------------------------------+
| Stellantis (整车制造与 OEM 底盘) |               | 单一实体 (如 Waymo / Tesla)      |
+----------------------------------+               |                                  |
                |                                  |  - 自研/定制硬件与传感器集成      |
                v                                  |  - 独立自研 AI 软件 Stack        |
+----------------------------------+               |  - 自营 Robotaxi 车队与 App      |
| Wayve (AI 算法 Stack 许可授权)   |               |  - 全自营的重资产运营            |
+----------------------------------+               +----------------------------------+
                |                                                   |
                v                                                   v
+----------------------------------+               +----------------------------------+
| Uber (出行需求网络与车队调度)    |               | 资本开支高昂、全栈掌控度高，     |
+----------------------------------+               | 多区域扩张速度受限               |
+----------------------------------+               +----------------------------------+
```

#### 1. 横向平台模型（Stellantis + Wayve + Uber）
* **资本效率极高：** Wayve 和 Uber 均无需承担造车的重资产资产负债表压力。Stellantis 负责 OEM 量产规模化；Uber 则借助既有的车队管理合作伙伴解决充电、清洁和驻车维护。
* **极速规模化与高资产利用率：** Uber 全球双边市场瞬间提供了极高的派单密度，极大地压缩了空驶率。高车辆利用率（以每小时产生营收的行驶里程衡量）是 Robotaxi 实现正向单车经济学（Unit Economics）的最核心变量。
* **软件 Licensing 模式：** Wayve 扮演了 Tier-1 软件供应商的角色，按车或按里程收取 AI 引擎授权费，享有典型的高毛利软件商业模式。

#### 2. 垂直整合模型（Waymo & 特斯拉）
* **Waymo：** 掌控从软件栈到自营打车 App（Waymo One）的全流程，同时向 OEM（捷豹、极氪）采购车辆并自行进行高昂的传感器改造。这种模式品控极高，但自行持有车队、重度改装和建设运力网点的 CapEx（资本支出）极其庞大。
* **特斯拉：** 试图通过定制 Cybercab、Unboxed 组装工艺、纯视觉 FSD 以及自营 Tesla Network App 打造终极垂直整合。如果成功，其利润空间巨大；但在缺乏专业车队运营伙伴的情况下，将面临巨大的硬件与法规摩擦。

#### 单车经济学指标对比：

| 指标维度 | AV 1.0 垂直整合模式 (Waymo 模式) | AV 2.0 横向平台模式 (Stellantis-Wayve-Uber) |
| :--- | :--- | :--- |
| **传感器硬件成本** | ~$30,000 – $70,000 / 车（高规格 LiDAR/雷达） | ~$5,000 – $12,000 / 车（以摄像头为主/原生 L4） |
| **测绘与城市扩充成本** | 极高（高精地图绘制与实时更新数千美元/英里） | 接近零（无图化零样本泛化） |
| **车队利用率与网络** | 自营 App（每进入一座新城市需重新建网获客） | 无缝接入 Uber 现有全球庞大需求网 |
| **预估每英里综合成本** | ~$1.50 – $2.50 / 英里 | 规模化目标 < $0.70 / 英里 |

### 行业论战与专家视点

无图基座模型与高精地图模块化架构之间的路线分歧，激化了硅谷与华尔街科技领袖们的激烈辩论。

#### **关于“无图端到端 AI”与“高精地图”的交锋：**

> *“AV 1.0 的本质是拟合规则去适应地图；而 AV 2.0 的本质是教 AI 学会像人类一样去驾驶。如果一个人仅凭视觉和智力就能在从未去过的异国城市顺畅开车，自动驾驶系统也必须做到这一点。高精地图只是一根限制规模化扩张的拐杖。”*  
> — **Alex Kendall**，Wayve 联合创始人兼 CEO
