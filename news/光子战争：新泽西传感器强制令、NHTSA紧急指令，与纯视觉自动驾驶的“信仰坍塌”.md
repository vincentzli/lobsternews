# 光子战争：新泽西传感器强制令、NHTSA紧急指令，与纯视觉自动驾驶的“信仰坍塌”

硅谷长期以来都奉行一个信条：软件可以解决任何硬件上的局限。然而就在今天，美国国家公路交通安全管理局（NHTSA）与新泽西州联手给自动驾驶（AV）行业浇了一盆冷水——两项协同落地的监管重拳，直指自动驾驶汽车感知物理世界的物理规律。

2026年7月8日，NHTSA局长乔纳森·莫里森（Jonathan Morrison）向所有自动驾驶开发商发出正式行动呼吁与指令，要求其在月底前针对一个顽固的安全故障提交具体的工程技术解决方案：无人驾驶车辆干扰应急救援。NHTSA记录了自动驾驶汽车闯入活跃救援现场、阻挡救护车和消防车，以及无法识别路障火炬（road flares）、交通路标锥和烟雾的“清晰模式”。该机构将这些失效定义为“功能性缺陷”（functional insufficiency）。

与此同步，由参议员安德鲁·兹维克（Andrew Zwicker）和戈登·约翰逊（Gordon Johnson）发起的美国新泽西州参议院第S1677号法案，在通过参议院交通委员会后，正在提交参议院预算和拨款委员会审议。S1677法案提出了一项为期三年的试点项目，其中包含条款对“纯视觉感知系统”堪称毁灭性打击：任何在该州运营的自动驾驶汽车，其传感器套件必须配备摄像头外加至少两种其他不同的感知模态（例如激光雷达LiDAR和雷达Radar）。此外，开发商必须在州内完成5万英里的配备安全员的监督测试，方可开启商业化运营。

这一双重监管威胁随即引爆了一场激烈的游说战。2026年6月，特斯拉向新泽西州的车主发起了大规模的公开动员，敦促他们反对S1677法案，声称该法案是“反竞争的偏袒行为”，制造了“任意的绊脚石”。这场争论已经超越了软件算法的范畴，演变成了一场围绕光物理学本质、量产成本结构以及联邦法律边界的全面战争。

### 应急救援危机：逼迫NHTSA出手的连环事故

NHTSA的这项指令并非防患于未然的警示，而是对堆积如山的事故报告的直接回应。其中最显着的转折点发生在2023年8月17日的旧金山，当时一辆Cruise无人驾驶Robotaxi在Turk街和Polk街交叉口与旧金山消防局（SFFD）的一辆消防车相撞。事发时，消防车正处于“Code 3”紧急状态（警笛长鸣且警灯闪烁），并驶入逆向车道以避开拥堵。尽管Cruise车辆检测到了这辆应急车辆，但其路径规划算法未能预测到消防车逆行跨车道的轨迹，最终导致碰撞，导致Robotaxi安全气囊弹出，一名乘客受伤住院。

包括国际消防员协会（IAFF）和旧金山消防局在内的应急救援组织长期以来一直指出，Robotaxi已成为威胁生命的公害。旧金山消防局记录了数十起“自动驾驶车辆干扰事件”，其中无人驾驶汽车碾压消防水带、堵塞消防车库门口，甚至在十字路口死机动弹不得，导致救护车无法及时运送重伤患者。

“如果救护车因为Robotaxi被闪烁的警灯搞糊涂而延迟两分钟，这很可能就是生与死的差别，”一位旧金山消防局代表表示。“光靠软件修修补补是解决不了问题的。我们需要物理层面的手动接管权限以及百分之百可靠的检测能力。”

### 感知的物理学本质：纯视觉为何在火光与烟雾中“抓瞎”

硬件路线之争的核心，在于以特斯拉为代表的“纯视觉（仅摄像头）”架构，与以Waymo、Cruise和Zoox为代表的“多传感器融合”方案之间的不可调和的矛盾。

要理解纯视觉系统为何在应急救援现场屡屡失效，我们必须回归电磁波谱与光散射的物理学本质。

```
可见光（摄像头）：     λ ≈ 380 nm – 740 nm （被动感知）
近红外（激光雷达）：   λ ≈ 905 nm / 1550 nm （主动感知）
毫米波（雷达）：      λ ≈ 3.9 mm (77 GHz) （主动感知）
```

#### 米氏散射 vs 瑞利散射

当光线穿过充斥着微粒的大气层（如雾、雨、雪或山火烟雾）时，会发生散射。决定这种散射数学特征的，是微粒直径（$D$）与电磁波波长（$\lambda$）的比值。

1. **米氏散射（$D \approx \lambda$）：** 雾滴（平均直径10至50微米）和雨滴（0.5至4毫米）的尺寸远大于可见光波长（0.4至0.7微米）和近红外激光雷达波长（0.9至1.55微米）。这使得它们直接落入米氏散射和几何反射的范畴。其结果是可见光和激光严重衰减、发生背向散射，并在激光雷达点云中产生“噪点”或“幽灵障碍物”；对于摄像头而言，则表现为对比度彻底丧失、强光眩目以及镜头遮挡。
2. **瑞利散射（$D \ll \lambda$）：** 毫米波雷达工作在77 GHz频段（波长 $\lambda \approx 3.9$ 毫米）。由于其波长远大于雾滴和细雨的尺寸，散射截面可以忽略不计（散射截面与波长的四次方成反比：$\sigma_s \propto \lambda^{-4}$）。因此，雷达信号能够以极低的衰减穿透恶劣天气、浓烟和沙尘。

```
       [大气微粒：雾/雨滴 (D)]
                       |
   波长 (λ)            |      散射效应
   --------------------+---------------------------------
   可见光              |      米氏/几何反射（高散射）
   近红外激光雷达      |      米氏/几何反射（高散射）
   77 GHz 毫米波雷达   |      瑞利散射（散射极小，可忽略）
```

在活跃的应急救援现场，消防部门通常会部署地面路障火炬（road flares）以及高亮度闪烁的LED警灯。对于纯视觉系统来说，这会诱发两个灾难性的失效：
*   **传感器饱和（高动态范围HDR失效）：** 高强度的闪烁LED警灯会导致摄像头像素瞬间饱和，产生局部光晕（blooming），使得神经网络丢失了用于估算深度所必需的纹理细节。
*   **热/光学羽流（Thermal/Optical Plumes）：** 地面火炬会释放出灼热而浓密的烟雾。摄像头在视觉上只能看到一个巨大的、低对比度的烟雾羽流，车辆的占用网络（Occupancy Network）可能会将其误判为坚硬的物理障碍物，导致车辆无预警急刹车；反之，如果算法被训练为忽略烟雾，它又可能会直接过滤掉真正的实体火炬以及其后方摆放的交通路标锥。

激光雷达可以提供直接的3D空间坐标映射（点云），并且不受光学眩光的影响，但在面对烟雾和水分吸收时依然有些力不从心（特别是在1550纳米波段，水分吸收达到峰值）。而雷达——特别是新型的高分辨率**4D成像雷达**——通过穿透烟雾和雨水直接测量距离、高度、方位角和多普勒速度，彻底避开了这些物理限制，为自动驾驶提供了一个纯视觉系统根本无法复制的“物理锚点”。

### 融合算法之争：传感器冲突 vs BEV Transformer

埃隆·马斯克曾多次为特斯拉的纯视觉方案辩护，声称多传感器套件会引入所谓的“传感器冲突”（sensor contention）。正如他在X（原推特）上写道：
> *“激光雷达和雷达实际上会因为传感器冲突而降低安全性。如果激光雷达/雷达与摄像头的数据发生冲突，应该听谁的？这种感知数据的模糊性会导致风险增加，而非减少。”*

马斯克的这个论点确实戳中了传统“后融合”（Late Fusion，又称目标级融合）架构的痛点。在后融合架构中，每个传感器运行自己独立的目标检测管线，车辆的中央计算平台试图使用启发式算法（如匈牙利匹配算法和卡尔曼滤波器）将各自生成的3D包围盒（bounding box）进行合并。如果摄像头误检测到了高速路牌的虚假反射，而雷达探测到了一个静止的金属板，系统就必须决定信任哪一个传感器。一旦决策失误，要么导致碰撞，要么引发危险的“幽灵刹车”。

```
[摄像头] ----> [目标检测 pipeline] ----\
                                      +--> [后融合合并器] --> [控制规划]
[雷达]   ----> [目标检测 pipeline] ----/      （存在传感器冲突风险！）
```

然而，现代自动驾驶工程早已超越了后融合时代。目前行业的事实标准是基于**鸟瞰图（BEV）Transformer**（例如BEVFormer或TransFusion）的**前融合/中融合**（Early/Mid Fusion）方案。

```
[多摄像头图像]   ----> [ResNet/Swin 骨干网络] ----\
                                                +--> [BEV Transformer 解码器] --> [3D包围盒]
[激光雷达点云]   ----> [PointPillars 骨干网络] --/     （交叉注意力机制融合）
```

在BEV Transformer架构中：
1. 原始的多摄像头图像特征与激光雷达/雷达的3D点云特征，首先通过神经网络骨干网络（backbones）被提取出来。
2. 随后，一个基于查询（query-based）的Transformer解码器通过交叉注意力（cross-attention）层，将这些多模态特征直接投影并融合成一个统一的3D体素网格（voxel grid）。
3. 网络直接在这个共享的空间中完成目标检测与占用流（occupancy flow）的预测。

这种机制彻底消除了传感器冲突，因为网络不再比对来自不同传感器的独立“意见”，而是在做检测决策之前，就已经在数学层面完成了底层特征的融合。如果某个摄像头的镜头被泥沙遮挡，交叉注意力机制的权重会自动将焦点转移到该区域对应的激光雷达和雷达特征上。

然而，这种数学上的优雅是以高昂的算力开销为代价的。运行多模态BEV Transformer需要车端配备算力恐怖的NPU（神经网络处理器）、极高规格的液冷系统，以及在工作频率各异的传感器（例如摄像头30Hz、激光雷达10Hz、雷达20Hz）之间实现极其严苛的时间戳同步（延迟匹配）。

### 经济账：量产车上的成本死穴

这种对于硬性传感器配置的监管倒逼，对消费级自动驾驶车辆来说无异于一堵高不可攀的“经济之墙”。

```
+--------------------------+---------------------+
| 传感器模态                | 估算单价（美元）     |
+--------------------------+---------------------+
| 车载摄像头                | $15 - $30           |
| 固态激光雷达              | $500 - $1,200       |
| 4D成像雷达                | $150 - $350         |
| 标准毫米波雷达            | $50 - $100          |
+--------------------------+---------------------+
```

一套像Tesla Vision那样的纯视觉系统，其原始硬件成本仅为150至200美元左右。相比制造端，一套鲁棒的多传感器套件（例如Waymo的第五代硬件栈，包含5个激光雷达、14个摄像头和6个雷达）单车成本高达1万至1.5万美元。

对于商业化运营的Robotaxi车队（SAE L4级）而言，这笔硬件成本可以在车辆的整个生命周期中轻易被折旧摊销，因为其替代的是最昂贵的人类司机成本。但是，对于消费级乘用车（SAE L2+ / L3级）来说，强制增加2,000至5,000美元的传感器硬件，将直接吞噬掉主机厂原本就极其微薄的利润空间。

如果新泽西州的S1677法案获得通过，它将开创一个危险的先例。一旦其他州纷纷效仿，美国的消费级自动驾驶市场将被彻底撕裂。一位在纽约州购车的车主，可能仅仅因为他的车采用了纯视觉系统，就被法律禁止驱车跨过特拉华河进入新泽西州。这必将倒逼整车厂要么彻底放弃纯视觉路线，要么不得不针对不同的州设计特供版的硬件配置。

### 宪法级对决：联邦预占权争夺战

一旦新泽西州正式实施S1677法案，该法律将立即面临联邦宪法层面的挑战。

根据美国《国家交通与机动车安全法》（联邦法典第49卷第30103(b)条），联邦政府在车辆安全标准上拥有明确的“预占权”（Express Preemption）：
> *“当本章规定的机动车安全标准生效时，任何州……仅在制定的标准与本章规定的标准完全一致的情况下，方可制定或继续实施适用于相同性能方面的标准。”*

从历史上看，双方的分工一直非常清晰：NHTSA负责监管车辆的*设计与安全性能*（通过联邦机动车安全标准FMVSS），而各州则监管这些车辆的*具体运营*（如驾照发放、交通法规、保险等）。

强制要求车辆必须搭载特定硬件（摄像头加另外两种模态），显然属于车辆设计和设备监管的范畴。自动驾驶行业——尤其是特斯拉——必将辩称新泽西州的法律企图规制车辆制造，因此已被联邦法律明确预占而失效。

然而，新泽西州的抗辩将立足于其自身的“警察权”（Police Powers）。该州会辩称，由于目前还没有一套专门针对自动驾驶系统（ADS）的全面联邦安全标准（FMVSS），各州为了保护本地道路使用者和应急救援人员，完全有权对其运营安全性进行规制。

正是这种法律上的模糊地带，让整个行业极度渴望类似于《自动驾驶法案》（SELF DRIVE Act）的联邦立法，以明确剥夺各州对自动驾驶汽车设计、构造或性能进行立法的权力。在此之前，仅需一个州的参议院法案，就足以让全美自动驾驶的商业化部署战略陷入法律的无底深渊。
